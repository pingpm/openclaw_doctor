工作原理

从崩溃到恢复
只需三步

1

运行安装命令

一条 curl | bash 命令自动安装所有依赖（Node.js、Cloudflare 二进制），并在端口 12222 启动 Doctor 服务，全程无需任何手动配置。

2

终端里出现公网地址

OpenClaw Doctor 使用 Cloudflare 内网穿透，无需路由器端口映射、无需域名备案，约 5 秒内自动生成形如 xxx.trycloudflare.com 的免费公网 URL。

3

把链接和 Token 发给 AI 或其他人

在任意浏览器打开链接（或让另一个 AI 智能体调用 API），即可获得一个实时终端，在远程电脑上执行任意命令：修配置文件、重启服务、查日志——和在本地操作一模一样。

功能特性

修好一台坏掉的
AI 智能体所需的一切

🌐

无需 SSH

只需一个 HTTPS 链接即可访问任意电脑。不需要用户名、密码，不需要开放防火墙端口，不需要配置公钥。

🔐

令牌保护

每次启动自动生成一个 128 位随机令牌。只有持有令牌的人或 AI 才能执行命令，无令牌访问一律返回 401。

🩺

OpenClaw 健康面板

自动运行 openclaw status 并将网关状态、活跃 Agent、会话数量、内存、安全警告以可读格式展示在侧边栏。

⚡

完整 Shell 权限

执行任意 Shell 命令、读写文件、列出目录——你在本地能做的，通过这个工具都能远程做到。

🤖

AI 智能体友好 API

标准 JSON REST 接口。任何 AI 智能体只需 POST /api/exec 一个命令，即可获得结构化的 stdout、stderr 和退出码。

🚇

Cloudflare 内网穿透

基于 Cloudflare 免费快速隧道服务，穿透 NAT、VPN 和防火墙，零配置——不需要账号，不需要域名。

API 文档

专为 AI 智能体设计

API · application/json · 请求头 x-doctor-token 或 ?token= 查询参数
# 存活检测（无需认证）
GET /api/ping
→ { "status": "alive", "app": "openclaw-doctor" }

# 获取完整的 openclaw status 输出
GET /api/status x-doctor-token: <token>

# 执行 Shell 命令
POST /api/exec x-doctor-token: <token>
请求体: { "cmd": "openclaw restart", "cwd": "/home/user" }
响应: { "stdout": "...", "stderr": "...", "code": 0 }

# 读取文件内容
GET /api/files/read?filePath=/path/to/file

# 写入 / 创建文件（自动创建目录）
POST /api/files/write
请求体: { "filePath": "/path/to/file", "content": "配置内容..." }

# 列出目录内容
GET /api/files/list?dirPath=/path/to/dir
